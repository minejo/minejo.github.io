1 <2015-10-16 周五>

本次会议主要讨论了项目近两个月来的工作重心，即发表一篇专利，目前专利重心分3个部分，一个是整个探测系统架构，一个是回波处理算法，一个是网络融合架构，经过讨论决定先发表探测系统架构的专利，由闫莉师姐牵头组织 相关事宜。

会上方老师提出了微波探测阶段需要注意的问题：

  1. 回波信号算法不能做成通用类型算法，必须以特定场景为依托，做特殊化适应，来体现创新新和实用性.
  2. 系统设计包括回波处理要充分考虑毫米波特性，以此为切入点考虑创新性.
  3. 信号处于毫米波频段，回波吸收问题需要考虑.

接着李超对已有的回波探测算法做了简要介绍，简要来说，通过线性调频连续波雷达的回波差频信号经过二维fft变换后得到物体速度和距离，再基于此得到 物体的运动轨迹，推测灾害类型。

之后微波探测项目的部分成员继续开了一个小会，主要讨论现阶段微波探测的任务以及任务划分。讨论结果如下：

  1. 扫描时是单波束，但是波束覆盖面广
  2. 扫描时是多波束，但是波束窄，符合毫米波特性，考虑波束切换时间（备用，先完善第一点）
  3. 预处理，处理杂波（何励励）
  4. 基于铁路特殊环境的杂波模型（（胥秋，刘贺）
  5. CFAR检测算法优化（刘伊敏）
  6. 物体运动轨迹获得（李超）
  7. ISAR获取物体大小（张迪）
  8. 多点模型

各个任务的时间周期等各位花两三天熟悉下再决定。

2 <2015-11-02 周一>

参会人员: 宋昊 闫莉 李超 何励励 刘伊敏 张迪 胥秋 刘贺

本次会议主要对探测组上两周的工作进度进行了一次汇报和总结，并制定了下一步的计划。

  1. 杂波抑制方向

    首先是刘贺和胥秋汇报的关于杂波抑制的方案，大概思路是先探测出静止杂波的回波，然后把有动物体的回波减去之前测的静止杂波的回波，理论上就可以 抑制杂波，具体细节不在此处论述。但是该方法由于线性调频连续波的波形使得该方案的性能不好，因而最后决定放弃该方案，另外寻找杂波抑制算法，预计两周时间。

  2. 杂波模型方案

    接着由何励励讲述杂波模型，基于均匀场景的杂波模型，如何模拟连续的杂波信号，但是场景是基于均匀场景，没有基于铁路方向的具体细化，而具体场景的杂波建模需要 采用DEM建模方法，需要实地测量，花费的时间和精力过大，暂时不予考虑。下一周工作主要完成杂波的多周期仿真，以及部分细节概念理清楚。

  3. CFAR

    刘伊敏先讲了CFAR的一维多策略算法以及仿真，效果还可以，然后提了2维cfar的算法。接下来两周重点考虑基于铁路灾情场景确定对应的二维多策略CFAR算法，争取这个方向 找到创新点。

  4. 获得运动物体轨迹

    李超讲了做的运动轨迹部分，主要讲了两种场景模型，二维和三维的。基于二维的模型，只考虑了距离和方位角，通过模拟雷达的波束扫描探测，经过多周期的 积累获得物体的运动轨迹。三维模型增加了俯仰角，可以直接获得物体在现实场景的三维轨迹，更加清晰直白，然后缺点是扫描周期变长了，可能会导致部分区域扫描不到。 目前场景模型和模型雷达探测模型已经建好，都是基于理想回波模型，需要结合其他部分才会看出效果，暂时没必要继续深入，下两周和刘伊敏一起研究 基于铁路灾害场景的CFAR算法。

  5. 获得运动物体大小形状

    张迪主要负责如何获得物体大小，具体原理和运动轨迹相似，如果有相邻的相似轨迹，可认为是一个物体，根据轨迹的横向距离判断物体大小。另外还提出了一个脉冲压缩的方案，但是需要修改下以适应到连续波雷达上，具体 方案和细节仍需一周-二周左右完善下

  6. 波束赋形

    宋昊主要从波束赋形的角度分析了下波束宽度，扫描周期，探测范围的博弈关系。基于其中一个参数为优化参数进行参数确定的博弈运算还需继续完善下。

3 <2015-11-17 周二>

参会人员：宋昊 闫莉 李超 何励励 刘伊敏 张迪 胥秋 刘贺

本次会议主要是对项目组上两周的工作进度进行了汇报和总结，并确定下一步方向。

本次做报告的主要有何励励的关于杂波模拟的报告，刘贺的关于杂波过滤预处理的报告，李超的关于cfar的报告， 闫莉的关于铁路微波承载网络关键技术报告。

  1. 杂波模拟

    何励励主要基于DEM数据建模方案模拟杂波，通过模拟随机地形从而得到随机杂波，暂时杂波以这个为标准做测试，等半实物板子测到真实杂波数据后再 比对杂波效果。

  2. 杂波过滤预处理

    刘贺主要介绍了基于MTI技术抑制杂波的方案，然后在MTI的基础上，采用MTD技术方案，优化抑制效果。同时也说明了该方案的主要缺陷：会产生一定的盲速。 即当运动目标的多普勒频率为f=n*fr时，这些运动目标也会被MTI滤波器抑制。

  3. CFAR算法

    李超首先介绍了一种基于二维变换后的CMLD-CFAR算法，通过在待测点所在行与列分别取一些点，然后排序并去除r个最大值后求平均，最后乘以系数k后与待测点 进行比较。然后展示了仿真结果，同时也讲了上次报告中svi-cfar算法的问题。

    主要问题如下：

    • 由于二维变化后，数据的特殊分布如果用常规cfar 检测方案会导致虚警率上升
    • CMLD-CFAR, SVI-CFAR方案仿真结果不是很理想，可能与一些参数的设置有关，仍需继续研究下

    同时，李超提出了基于二维数据特性的优化cfar算法的方案，即可以考虑多重判断，通过判断相邻多行数据是否有相同 的尖峰点来过滤虚警点.

  4. 毫米波融合架构的信令算法

    闫莉主要对铁路微波承载网络架构，铁路微波回传方案进行了介绍。

    1. 铁路微波承载网络架构

      主要介绍了基于云的探测与通信融合网络架构，低频RRU采用全向覆盖，负责可靠数据传输；毫米波RRU采用定向波束，负责列车通过时大容量数据传输及无列车时的环境探测。

    2. 铁路微波回传方案
      • 毫米波RRU通信发现。主要介绍了毫米波RRU如何进行探测和通信的切换。可以借助低频RRU当前切换或随机接入请求情况及列车位置信息判断列车是否要驶入当前毫米波RRU覆盖范围。若有列车驶入，停止探测做通信；可以只停止当前毫米波RRU的探测，其他相邻毫米波RRU仍做探测，可能会对通信造成干扰；或者停止该低频RRU覆盖范围内所有毫米波RRU的探测。列车驶出该低频RRU覆盖范围后（结合列车位置信息及当前切换请求判断）毫米波RRU停止通信做探测。
      • 低频RRU与毫米波RRU通信协作。主要介绍了低频RRU与毫米波RRU功能上的独立及协作该怎样设计。
      • RRU PHY。主要介绍了根据高铁场景的一些特殊性，比如用户数稀疏、运动速度快、快时变、多径稀疏，选择合适的多址及调制方式，如单载波更适合。参考信号的分布也应该根据该场景进行特殊设计。
  5. 下一步工作分配
    • 李超、何励励、刘伊敏主攻cfar方向
    • 刘贺、张迪继续完善杂波滤除算法，同时利用已经生成的杂波检验杂波滤除算法有效性
    • 胥秋先去帮忙调仿真板，尽快调通

4 <2015-11-30 周一>

参会人员： 李恒超 宋昊 闫莉 李超 何励励 刘伊敏 张迪 胥秋 刘贺

本次会议主要对上两周工作总结与汇报，同时对项目开题报告的反馈进行讨论。

本次做报告的有刘伊敏的关于高频未授权频段的调研，刘贺的关于杂波抑制的算法，李超的关于cfar算法的改进

  1. 高频未授权频段调研

    主要介绍了高频频段使用的情况和与场景结合时应注意的事项，比如当频段是某一个范围时大气损耗特别严重，这时应避免使用该频段

  2. 杂波抑制算法

    主要方式还是采用脉冲压缩，mti与mtd的工作方式，工作体系还是脉冲雷达，如何与连续波雷达相互兼容还需要进一步研究。

  3. cfar算法改进

    通过二维的cfar，即距离向与多普勒向的cfar，综合判断是否有物体出现，然后加入优化算法，即判断该点是否为尖峰点，最后cfar效果还可以，下一步需要解决邻近物体的cfar判断问题。

    本次开会还对开题报告时的反馈进行了讨论。

    下一步可以先考虑下累积算法与稀疏算法在本项目中的有效性。

    同时李恒超老师指出对于不同的场景可以使用不同的雷达工作方式，不一定非要把所有的功能糅合到一起，比如连续波雷达在异物入侵这块可能性能好一点，脉冲雷达在ISAR成像监测滑坡这块性能好一点。

  4. 下一步工作分配

    考虑到研三亟需把半实物平台搭建出来，因而胥秋和刘贺转到半实物组，优先考虑把半实物板子调试成功。

    雷达算法优先考虑使用连续波的工作方式解决铁路场景下的异物入侵问题，争取以这个场景发论文或者专利。

5 <2015-12-09 周三>

参会人员： 宋昊 闫莉 李超 何励励 刘伊敏 张迪 胥秋 刘贺

本次会议主要对异物侵限应用上的整合和使用连续波雷达成像方案的可能性。

首先张迪从连续波雷达成像可行性研究入手，分别介绍了

  • 脉冲SAR与FMCW SAR系统区别
  • FMCW SAR原理
  • FMCW SAR点目标回波模型

最后得出FMCW-ISAR方案在项目中具有可行性，可先往这个方案研究，如果效果可以则加入到项目方案中。

然后李超首先介绍了下常见的3种铁路异物侵限监测场景和几种常见异物侵限类型，然后介绍了已有的几种监测雷达在异物侵限中的应用，并从中找到借鉴之处。 然后基于自身项目，提出了对于这3种场景可采用的方案，该方案的最关键问题在于波束如何准确定位到正确的位置。 然后提出了轨迹分析方案，主要分为3个小点

  • 单目标情况下如何有效利用轨迹和速度信息
  • 多目标情况下如何在多点的情况下确定一个目标的轨迹（相关赋值与航迹相关）
  • 如何利用多目标轨迹确定物体大小，推测物体类型
  • 根据推测类型判断预警危险级数

本月剩余的时间除了继续研究外，需要完成项目年度研究报告，并提交铁总审核。

6 <2015-12-28 周一>

参会人员： 方旭明 宋昊 李超 何励励 刘伊敏 胥秋 刘贺

本次开会主要对本年度项目问题的总结与下一步工作方向的探讨。

首先方老师针对合同要求重新说明了下本项目的工作重心，要求实现整体创新，项目方案应该是一个整体，且为了使成果具有可竞争性，项目方案应该实现小而精，应该复杂度低，成本低，不能过分追求精确度而忽略复杂度。同时方老师讲了高铁相关的应用场景，寻求把项目适应于高铁轨道检测的方案。

接着李超主要对项目迄今为止的主要问题进行了讨论，主要如下：

  1. 窄波束之间不能区分物体的问题
  2. 天线选型问题还没有研究
  3. 扫描周期时长的协调问题
  4. 未授权频段的管制政策
  5. 仿真杂波的真实性验证

同时讨论了下一步的工作重心：

  1. 二维上研究多目标CFAR，争取出一篇论文
  2. ISAR方案，看能否整合（应用积累，稀疏方案）
  3. 半实物仿真，验证本年度算法
  4. 连续波波形设计、杂波消除、轨迹分离与分析
  5. 轨迹多目标、轨迹曲线模式匹配
  6. 融合架构包括警报帧设置、具体警报流程、交互流程

7 <2016-03-16 周三>

参会人员： 方旭明 李恒超 宋昊 李超 何励励 刘伊敏 张迪 胥秋 刘贺

本次开会主要是对项目过去一个多月的工作成果的汇报并讨论。

首先方老师对项目的背景进行了强调，在保证基站通信效果的前提下设计雷达探测算法。由于雷达探测与通信基站结合在一起，考虑到成本原因，雷达的 距离分辨率，方位分辨率等各项指标不会高，如何提出一个结合实际铁路场景的低成本的雷达探测与预警方法应该是工作重心。

接着，何励励讲了二维CFAR算法的仿真，分析了三种算法的性能CA-CA,CA-OS,OS-OS，由于十字窗用法是基于经验得到的，并没有理论推导，所以在相关概念上还比较模糊，还需要在讨论讨论。 同时，该算法相对较简单，对于大规模的物体出现时候的场景能不能检测出来还有待考验。

刘伊敏主要讲了二维CFAR算法的多策略选择方案，其实就是一维多策略方案直接搬到了二维上，窗口的选择和两个方向的空间、时间相关性还没有考虑。下一步工作可以考虑先仿真试试，单目标，多目标，然后再尝试下结合实际场景降算法复杂性。

李超首先就是否有必要对每个单独目标进行跟踪的问题进行了讨论。对每个目标进行跟踪的话也就是常规的雷达跟踪算法，普通场景下相对可以保证每个物体的运动轨迹分离出来，但是对于泥石流等大规模物体出现的场景，该算法性能就比较差或者实现的相对成本比较高。 而且获取轨迹的初期目标是为了辅助判断出现的点迹是否是异物入侵，是否有威胁。方老师的意思是把同一时刻返回的点迹当做一个整体来判断，通过多周期的积累判断整体运动的趋势，如果接近轨道则预计，不关心具体的单个轨迹和物体属于的类别。同时，李老师提出对于虚警问题，可以通过周期 积累的方法解决虚警的问题。

接着李超讲了智能波束扫描的方案，首先用宽波束全区域扫描，扫到目标后针对每个目标建立跟踪文件，分时在每个目标周围用窄波束扫描，并整体轨迹维持，到达特定时间后再用宽波束扫全部区域，如此循环，该方案可缓解因方位向分辨率不足导致的方位跳变问题。 李老师由此提出一个类似通信跳频的方案，通过将铁路待探测区域分区，雷达使用多波束技术，不同的波束使用不同的频段，高频频段扫描近区域，分辨率较高，连续有物体就报警，相对低频频段扫描远区域，分辨率相对较低，但是缓解了高频频段远距离传输困难的问题，通过判断有整体轨迹接近轨道的趋势就报警。

接着张迪介绍了下半实物板的最新进展，展示了他们用连续波ISAR方案使用单波束检测出远处汽车的实验结果。

对于下一步规划：

  1. 主要针对以上探测方案，提取出创新点，发一篇专利，最好再出一篇论文。
  2. CFAR需要考虑下多目标的情形，以及参考窗选择的逻辑问题
  3. 轨迹这块完善新的方案，并仿真出结果
  4. ISAR从理论到仿真，出一套整体方案
